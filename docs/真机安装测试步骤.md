# iPhone/iPad 真机安装测试步骤

> 本文档所有命令均已在 2026-07-19 实测跑通（iPhone 17 Pro Max，Xcode 26.5 SDK，Team R59Q7NXAWD）。
> 适用场景：把当前代码构建成开发签名的 App 装到自己的 iPhone/iPad 上做真机测试。
> 与 TestFlight 的区别：本方式零等待、随改随装，但**开发签名 7 天过期**（过期后 App 打不开，重装即可）；TestFlight 版 90 天有效且可分享他人（见 P4-10，另行搭建）。

---

## 一、一次性准备（新 Mac / 新手机 / 新账号才需要）

已经在本机做过的可跳过；换电脑或换手机时按序补齐。

1. **Apple Developer 账号**：付费开发者账号，Xcode → Settings → Accounts 里登录对应 Apple ID。
2. **开发证书**：同页面 → 选中账号 → Manage Certificates… → ＋ → Apple Development（钥匙串弹窗点允许）。
3. **签名团队**：`src-tauri/tauri.conf.json` 的 `bundle.iOS.developmentTeam` 已配置为 `R59Q7NXAWD`；换团队时改这里。
4. **手机侧**：
   - 数据线连 Mac → 手机解锁 → 弹窗点「信任此电脑」
   - 设置 → 隐私与安全性 → 开发者模式 → 打开并重启（此开关可能在首次尝试安装后才出现）
5. **⚠️ 全新环境的第一次部署必须走一次 Xcode 图形界面**（实测教训：命令行 xcodebuild 拿不到 Xcode 的账号会话，无法自动创建描述文件）：
   - `open src-tauri/gen/apple/shelf.xcodeproj`
   - 顶部设备选真机 → TARGETS shelf_iOS → Signing & Capabilities 确认 Team → ⌘R
   - 手机弹「不受信任的开发者」→ 设置 → 通用 → VPN与设备管理 → 信任你的开发者账号
   - 这一步会自动完成：注册设备到描述文件、创建 provisioning profile、挂载开发镜像。之后命令行方式即可独立工作。

## 二、日常安装（命令行三步，推荐）

```bash
# 0. 查设备 UDID（连接并解锁手机后执行；取第三列那串带横杠的 ID）
xcrun devicectl list devices

# 1. 构建开发签名 IPA（全量编译约 5-10 分钟）
npm run tauri ios build -- --export-method debugging
# 产物固定在：src-tauri/gen/apple/build/arm64/Shelf.ipa

# 2. 安装到手机（把 <UDID> 换成第 0 步查到的）
xcrun devicectl device install app --device <UDID> src-tauri/gen/apple/build/arm64/Shelf.ipa

# 3. 启动（也可以直接在手机上点图标）
xcrun devicectl device process launch --device <UDID> com.shelf.reader
```

实测参考输出：安装完成后 `xcrun devicectl device info apps --device <UDID> | grep -i shelf` 应列出 `Shelf com.shelf.reader 0.2.2`。

## 三、热调试模式（改前端代码实时生效，可选）

开发 UI 时不想每次重装，用 dev 模式跑真机（前端改动经 Wi-Fi 热更新，Rust 改动仍需重跑）：

```bash
npm run tauri ios dev --host   # 选择真机；手机和 Mac 必须在同一 Wi-Fi
```

注意：dev 模式的 App 依赖 Mac 上的 vite 服务，拔线/关终端后 App 打不开属正常；正式测试请用第二节的 IPA 安装。

## 五、常见问题（全部为实际踩过的坑）

| 现象 | 原因与解法 |
|---|---|
| `No Account for Team` / `No profiles for 'com.shelf.reader'` | 命令行拿不到 Xcode 账号会话。按第一节第 5 条走一次 Xcode GUI，之后命令行恢复正常 |
| `ld: library 'clang_rt.iossim' not found` 或指向旧 clang 版本路径 | Xcode 升级后构建缓存过期：`rm -rf src-tauri/target/aarch64-apple-ios*` 后重建 |
| 手机上 App 图标点了闪退/提示不受信任 | 设置 → 通用 → VPN与设备管理 → 信任开发者 |
| 装了 7 天后打不开 | 开发签名过期，按第二节重装即可（数据不丢） |
| `devicectl` 里设备显示 `unavailable` 或 `no DDI` | 手机未解锁/未信任/开发者模式没开；处理后重插数据线 |
| dev 模式 App 白屏 | 手机与 Mac 不在同一 Wi-Fi，或 vite 端口 1420 被占：`lsof -ti :1420 \| xargs kill -9` 后重跑 |
| 构建报签名残留错误 | 参见 fixedbug.md「Team 签名残留」条目：gen/apple 工程里的旧 Team 需在 Xcode Signing 页重选 |

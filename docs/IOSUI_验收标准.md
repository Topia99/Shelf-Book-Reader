# iOS 阅读器壳层改造——审核意见与验收标准

上游计划：[IOSUI_design.md](../IOSUI_design.md)
审核结论：**方案通过**（架构方向正确：不换 PDF 内核、壳层隔离、样式命名空间、缩略图窗口化渲染）。
以下四处修订为硬性要求，与分阶段验收标准一并构成交付定义。执行者按本文档逐条自查后交付。

---

## 一、审核修订（必须遵守）

### 修订 1：本批范围收窄——只做已有能力的新外壳，不引入新功能

- 顶栏右侧 pill 第一批**只放已有能力**：缩放模式切换（适宽/整页）。
- **搜索、书签、外观/主题、字号**：本批一律不做、不放占位按钮（空按钮点了没反应是更差的体验）。设计文档中的"功能优先级 6"整体移到下一批。
- 结果：第一批 iOS 顶栏 = 左 pill（返回、目录）+ 右 pill（缩放模式）+ 页码 badge + 底部缩略图 strip。

### 修订 2：平台判断必须能正确覆盖 iPad

- `navigator.userAgent` 里 iPadOS 会伪装成 "Macintosh"，纯 UA 判断会把 iPad 错路由到桌面壳。
- 要求 `src/platform.ts` 新增 `isIos`，判定逻辑必须是：
  `/iPhone|iPad|iPod/.test(ua) || (ua 含 "Macintosh" && navigator.maxTouchPoints > 1)`
- iPhone 与 iPad **都路由到 `ReaderIosShell`**；Windows/macOS（含触屏 Windows 笔记本）路由到 `ReaderDesktopShell`。触屏 Windows 不得误判为 iOS。

### 修订 3：缩略图必须纳入渲染内存预算（P4-3 的延伸）

- 缩略图渲染宽度 ≤ 150 物理像素（不乘 dpr 超过 2）。
- 渲染完成后立即 `canvas.toDataURL("image/jpeg", 0.7)` 转成图片字符串并销毁 canvas，DOM 里的 strip 用 `<img>`，不保留活 canvas。
- 缓存上限 24 张，LRU 淘汰；换书时清空。
- 缩略图渲染任务必须排在主页面渲染之后（串行队列或空闲时机），翻页时立即取消未完成的缩略图任务，不得让主页面渲染排队等缩略图。

### 修订 4：共享核心的行为清单——拆分不是重写

从 Reader.tsx 提取 `usePdfReaderController` + `ReaderPageStage` 时，以下现有行为必须原样保留在共享层（两个壳都吃到）：

1. 页面渲染：可见页渲染、双页模式、渲染分辨率预算（dpr≤2 + 1600 万像素上限，P4-3 的逻辑一行不动）
2. 触屏手势：左右滑翻页、点击两侧翻页区、捏合缩放、长按取词（P4-4 的逻辑）
3. 键鼠：方向键/空格/PageUp 翻页、修饰键+滚轮缩放、滚轮到边翻页（桌面壳用）
4. 取词弹窗（WordPopup）与文字层
5. 目录数据加载与跳转
6. 进度保存与防抖（beforeunload flush + update_progress 触发同步）
7. 加密 PDF 密码输入回退界面

## 二、分阶段验收标准

### Phase 1：结构拆分（纯重构，行为零变化）

| # | 验收项 | 判定方法 |
|---|---|---|
| 1.1 | 新增 `src/components/reader/` 下的 controller/stage/desktop shell 三个文件，`Reader.tsx` 变为薄路由层或被替换 | 代码审查 |
| 1.2 | `npx tsc --noEmit` 零错误 | 命令 |
| 1.3 | 桌面端（macOS `tauri dev`）过一遍第三节回归清单，全部与改造前一致 | 人工 |
| 1.4 | 本阶段 **不引入任何视觉变化**（截图对比改造前后一致） | 截图对比 |

### Phase 2：平台路由

| # | 验收项 | 判定方法 |
|---|---|---|
| 2.1 | `platform.ts` 的 `isIos` 按修订 2 实现，且有注释说明 iPad UA 伪装问题 | 代码审查 |
| 2.2 | iOS 模拟器（iPhone）进阅读页 → `ReaderIosShell`；macOS → `ReaderDesktopShell` | 各平台实跑 |
| 2.3 | 路由错误时的兜底：`isIos` 判定失败不会白屏（默认落桌面壳） | 代码审查 |

### Phase 3：iOS 壳层 UI

| # | 验收项 | 判定方法 |
|---|---|---|
| 3.1 | 顶部两组悬浮 pill：左（返回、目录）右（缩放模式），毛玻璃底、圆角胶囊、避开安全区（`--safe-top`） | 模拟器截图 |
| 3.2 | 页码 badge 独立悬浮（如 `158 / 246`），翻页实时更新，工具栏隐藏时 badge 仍短暂可见后淡出 | 模拟器操作 |
| 3.3 | 点击页面中央区域切换工具栏显隐，动画 ≤300ms，隐藏时全屏沉浸（现有 P4-4 行为接入新壳） | 模拟器操作 |
| 3.4 | 样式全部使用 `.reader-ios-*` 命名空间，`git diff` 中现有桌面类（`.reader-toolbar` 等）零改动 | diff 审查 |
| 3.5 | iOS 默认缩放为 fit-width；页面舞台背景用柔和浅灰（与书库白色主调一致） | 模拟器截图 |
| 3.6 | 竖屏/横屏旋转后布局不破（pill 不与刘海/圆角重叠，strip 不被 home indicator 遮挡） | 模拟器旋转 |
| 3.7 | 取词、目录跳页、进度保存在 iOS 壳内全部可用 | 模拟器操作 |

### Phase 4：底部缩略图 strip

| # | 验收项 | 判定方法 |
|---|---|---|
| 4.1 | strip 显示当前页 ±5 页窗口，横向滑动，当前页赭色描边高亮 | 模拟器操作 |
| 4.2 | 点缩略图跳页，跳页后窗口跟随移动 | 模拟器操作 |
| 4.3 | 修订 3 的全部内存条款（150px 上限 / dataURL 化 / 24 张 LRU / 主渲染优先） | 代码审查 + 实测 |
| 4.4 | 300MB 扫描版 PDF 上开 strip 连续滑动翻页，App 不崩、主页面渲染无可感知卡顿 | 真机/模拟器实测 |
| 4.5 | strip 随工具栏一起显隐 | 模拟器操作 |

## 四、通用交付门槛（每个 Phase 提交前）

1. `npx tsc --noEmit` 零错误
2. `cargo clippy -- -D warnings` 零警告、`cargo test` 全绿（本改造理论上不碰 Rust，若碰了必须说明原因）
3. 推送后 GitHub CI 三 job 全绿
4. 每个 Phase 一个独立 commit，信息写明该阶段验收项完成情况
5. progress.md 同步更新（本改造在看板中挂为 P4-UI-1 ~ P4-UI-4）

## 五、明确不做（防止范围蔓延）

- 搜索、书签、外观主题、字号调节（下一批）
- 全书缩略图网格浏览（只做窗口式 strip）
- 桌面端任何视觉调整
- PDF 内核替换、EPUB

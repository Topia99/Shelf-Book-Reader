# Shelf — 本地 PDF 书库阅读器

把散落在文件夹里的 PDF，变成一个有封面、有进度、开箱即读的个人书架。

桌面应用，体验对标 iOS「图书」App：添加书籍后所有阅读都在软件内完成，
书库以封面墙展示，记住每本书的阅读进度，点开即回到上次读到的那一页。

![技术栈](https://img.shields.io/badge/Tauri%202-React%20%2B%20TypeScript-blue)
![平台](https://img.shields.io/badge/Windows-10%2F11-brightgreen)

当前发布版主要面向 Windows；仓库已包含 Tauri 的 macOS 开发运行配置，可在 MacBook 上本地启动调试。

## 功能

- **复制入库**：添加 PDF 时复制到软件专属书库，源文件之后可自由移动或删除；SHA-256 查重，同一文件改名再添加也能识别
- **封面书架**：自动提取第一页做封面，支持按最近阅读 / 添加时间 / 书名排序，书名搜索
- **进度记忆**：翻到哪记到哪，重开软件自动回到上次读的那一页（强杀进程最多丢失 1 秒进度）
- **阅读器**：单页 / 双页视图、适合页宽 / 适合整页 / 自由缩放、目录侧边栏、页码跳转
- **加密 PDF**：支持输入密码打开
- **单词即时翻译**（v0.2.0）：读英文书时双击任意单词，原地弹出中文释义、音标与发音，
  内置 36 万词条离线词典（基于 ECDICT），变形词自动还原（went → go），全程无需联网
- 全部数据保存在本地 `%APPDATA%\Shelf`，无账号、无云、无网络请求

## 安装

1. 到 [Releases](../../releases) 页面下载最新的 `Shelf_x.x.x_x64-setup.exe`
2. 双击运行，一路「下一步」即可（安装到当前用户，不需要管理员权限）
3. 安装完成后桌面会出现 Shelf 图标，双击启动

> 卸载：Windows 设置 → 应用 → Shelf → 卸载。书库数据在 `%APPDATA%\Shelf`，如需彻底清除可手动删除该文件夹。

## 使用

- **添加书籍**：点击右上角「＋ 添加书籍」选择文件（可多选），或直接把 PDF 拖进窗口
- **打开阅读**：双击封面
- **重命名 / 移除**：右键封面（移除只删除书库内副本，不影响你的源文件）
- **查单词**：阅读中双击英文单词即弹出释义；点 🔊 朗读；点弹窗外、按 Esc 或翻页关闭

### 阅读器快捷键

| 按键 | 功能 |
|------|------|
| `←` `→` / `PageUp` `PageDown` / 空格 | 翻页 |
| 鼠标滚轮（页面边缘）/ 点击画面左右两侧 | 翻页 |
| `Ctrl` + 滚轮 | 缩放 |
| `Home` / `End` | 跳到第一页 / 最后一页 |
| 页码框输入数字回车 | 跳转指定页 |
| `Esc` | 返回书库 |

## FAQ

**Q: 添加书籍后源文件还需要保留吗？**
不需要。入库即复制，之后源文件随便移动、删除都不影响书库。

**Q: 支持 EPUB / 深色模式 / 高亮标注吗？**
暂不支持，都在后续版本计划中。

**Q: 取词翻译为什么查不到某些词？扫描版能取词吗？**
词典为离线裁剪版（约 36 万词条），极生僻词可能未收录。扫描版（纯图片）PDF 没有文字层，
暂不支持取词，双击时会提示。安装包因内置词典比 v0.1.0 增大约 15 MB。

**Q: 安装时 Windows 提示「已保护你的电脑」（SmartScreen）？**
因为安装包没有代码签名（个人项目未购买证书）。点「更多信息」→「仍要运行」即可。

**Q: 我的书库数据存在哪里？**
`%APPDATA%\Shelf`：`library.db`（书目与进度）、`books/`（PDF 副本）、`covers/`（封面缓存）。

## 开发

技术栈：Tauri 2 + React + TypeScript + PDF.js + SQLite（rusqlite）

### 在 MacBook 上启动

1. 进入项目目录：

```bash
cd shelf-book-reader
```

2. 安装前端依赖：

```bash
npm install
```

3. 确认已安装 Rust 工具链；如果没有，先执行：

```bash
curl https://sh.rustup.rs -sSf | sh
```

4. 启动 Tauri 的 macOS 开发版：

```bash
npm run tauri dev
```

5. 等待编译完成后，`Shelf` 的原生 macOS 窗口会自动打开。

补充说明：

- 前端开发服务运行在 `http://localhost:1420`
- 真正使用的是 Tauri 拉起的原生 macOS 应用窗口，不是单独的浏览器页面
- 如需生成 macOS 安装产物，可执行 `npm run tauri build`

### 通用开发命令

```bash
npm install        # 安装前端依赖（需要 Node.js 与 Rust 工具链）
npm run tauri dev  # 开发模式运行
npm run tauri build  # 打包 NSIS 安装程序
```

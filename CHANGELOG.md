# CHANGELOG

## v0.3.0 - 2026-07-22

从「Windows 单机阅读器」跃迁为「全平台阅读器」的大版本。

### 新增
- **全平台**：macOS（Apple Silicon）、iOS / iPadOS（TestFlight）与 Windows 同一套代码，触屏端专属阅读壳（悬浮工具栏、滑动/点击翻页、捏合缩放、长按取词、底部缩略图胶片）
- **账号 + 云端同步（可选）**：登录后书目与阅读进度在多设备间自动同步；本地优先，不登录即完整单机使用（后端 Supabase 行级隔离 + Cloudflare R2）
- **「极简白」UI 重设计**：首页只留书，搜索/排序/账号收进 ⋯ 菜单，参照 Apple「图书」
- **移动端文件导入**：从其他 App「分享到 Shelf」或「用 Shelf 打开」PDF
- **生命周期持久化**：移动端切后台即刻落盘，防进度丢失

### 优化
- 离线取词词典在全平台可用（iOS 实测词形还原/后缀命中）
- 数据目录跨平台化（相对路径存储，随容器迁移不失效）
- iOS 阅读页满宽渲染修复

### 分发
- macOS DMG / Windows 安装包经 GitHub Actions 构建并发布到 Releases
- iOS 经 GitHub Actions 自动上传 TestFlight（推 `v*` tag 触发）

## v0.2.2 - 2026-07-05

### 修复
- 双击取词选中的词与视觉位置不符（如双击 end 弹出 hello）：
  html 的 lang="zh-CN" 使选词层的通用 sans-serif 被解析为中文字体，拉丁字母比
  PDF.js 测量基准（Arial）宽约 12%，横向逐字累积漂移。为文字层标注 lang="en"
  使字体解析与测量一致。已通过 CDP 在真实应用内截图对比验证红黑文字完全重合

## v0.2.1 - 2026-07-05

### 修复
- 双击取词完全无响应：全局 `user-select: none` 连带禁掉了 PDF 文字层的双击选词，
  导致选区始终为空。文字层单独放开 `user-select: text` 后恢复正常
- 查词请求失败时给出 toast 提示（原为静默失败）

## v0.2.0 - 2026-07-05

### 新增：单词即时翻译（取词弹窗）
- 阅读英文 PDF 时双击单词，原地弹出翻译卡片：单词、音标、中文释义（按词性分行，
  超 5 条折叠）、发音按钮（系统合成语音，离线）
- 变形词自动还原（running → run、went → go），标注原形
- 内置离线词典：ECDICT 裁剪版 36.6 万词条 + 5.5 万词形反查，dict.db 23.8 MB，
  作为只读资源随安装包分发，与业务库 library.db 物理分离
- 边界处理：首尾标点自动剥离、大小写归一、中文/数字/多词不响应、
  生僻词显示"未找到释义"、扫描版 PDF 提示不支持（每次会话仅提示一次）、
  无英文语音时隐藏发音按钮
- 关闭方式：✕ / 点击外部 / Esc / 翻页缩放滚动自动关闭；新弹窗替换旧弹窗
- 技术：PDF 页面新增透明文字层（PDF.js TextLayer）承载双击选词；
  点击翻页区改为基于点击位置判定（文字上的点击留给选词，不触发翻页）

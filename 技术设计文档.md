## 1. 项目目标（Objectives）

**一句话定位**：把散落在文件夹里的 PDF，变成一个有封面、有进度、开箱即读的个人书架。

| 目标 | 说明 | 达成状态 |
|------|------|----------|
| 一站式阅读 | 添加书籍后，所有阅读在软件内完成，不再依赖文件夹 + 外部阅读器 | ✅ v0.1.0 |
| 复制入库 | 对标 iOS「图书」：入库即复制，源文件可自由移动/删除 | ✅ |
| 进度记忆 | 每本书记住读到哪页，重开软件回到上次位置，强杀进程最多丢 1 秒 | ✅ |
| 封面书架 | 自动提取第一页做封面，网格展示、排序、搜索 | ✅ |
| 零门槛分发 | 3 MB 安装包，免管理员权限，装完自动出桌面图标 | ✅ |
| 本地优先 | 无账号、无云、无网络请求，数据全部在本机 | ✅（长期原则） |

**明确不做（MVP）**：深色模式、书签/高亮标注、阅读统计、EPUB、云同步、文件关联、代码签名、自动更新 → v1.1+ 待办池。

## 2. 技术栈

| 层 | 技术 | 版本 | 用途 |
|----|------|------|------|
| 应用框架 | Tauri | 2.11 | 桌面壳（WebView2）、IPC、打包 |
| 前端框架 | React + TypeScript | 18.3 / 5.6 | 书库与阅读器 UI |
| 构建工具 | Vite | 6 | 前端开发服务器与打包 |
| PDF 渲染 | PDF.js（pdfjs-dist） | 4.10 | 页面渲染、元数据、outline、封面提取 |
| 后端语言 | Rust | stable (MSVC) | 文件操作、哈希、数据库 |
| 数据库 | SQLite（rusqlite, bundled） | 0.32 | 书目与阅读进度，WAL 模式 |
| 哈希 | sha2 | 0.10 | 文件 SHA-256 查重 |
| 系统对话框 | tauri-plugin-dialog | 2 | 文件多选 |
| 安装包 | NSIS（Tauri bundler） | — | currentUser 免管理员安装 |

**选型理由**
- **Tauri 而非 Electron**：安装包 3 MB vs 上百 MB；Rust 侧天然适合文件/哈希/SQLite 这类小而关键的原生任务；Windows 11 自带 WebView2 零依赖。
- **PDF.js 在前端渲染**：避免 Rust 侧引入 PDF 库；封面提取复用同一渲染管线（canvas → PNG）。
- **rusqlite 而非 tauri-plugin-sql**：业务接口（commands）本就定义在 Rust 侧，直接持有连接更简单，且便于做 WAL、事务等细粒度控制。

## 3. 架构设计

```
┌─────────────────────────── Tauri 窗口 ───────────────────────────┐
│  React 前端                                                       │
│  ├─ Library.tsx  书库网格 / 搜索排序 / 拖拽入库 / 右键菜单          │
│  ├─ Reader.tsx   PDF.js 渲染 / 翻页缩放 / 目录 / 进度节流           │
│  └─ pdf.ts       PDF.js worker / 封面提取（canvas → PNG bytes）    │
│        │ invoke (IPC)                  │ http://asset.localhost    │
│        ▼                               ▼ (Range 流式读取)          │
│  Rust 后端（lib.rs）                asset 协议白名单                │
│  ├─ commands：add_books / list_books / remove_book /              │
│  │            update_progress / rename_book / save_cover / …      │
│  ├─ SHA-256 流式计算（1 MB 缓冲）                                  │
│  └─ rusqlite（WAL） ──► %APPDATA%\Shelf\library.db                │
└───────────────────────────────────────────────────────────────────┘
```

**职责分工原则**：前端负责一切「看得见」的（含 PDF 渲染与封面提取）；Rust 只做文件复制/删除、哈希、SQLite 读写，代码量刻意保持很小（单文件 lib.rs）。

**大文件策略**：PDF 不经 IPC 传输，前端通过 Tauri asset 协议以 HTTP Range 流式读取书库文件，500 MB 书籍不会整体载入内存。

## 4. 数据模型

```sql
CREATE TABLE books (
  id             INTEGER PRIMARY KEY,
  hash           TEXT UNIQUE NOT NULL,   -- SHA-256，查重键 & 文件名
  title          TEXT NOT NULL,          -- 默认 PDF 元数据 Title，空则文件名
  file_path      TEXT NOT NULL,          -- 书库内副本路径
  cover_path     TEXT,                   -- NULL = 用「纯色+书名」默认封面
  total_pages    INTEGER,                -- 首次成功打开时回填
  current_page   INTEGER NOT NULL DEFAULT 1,
  added_at       TEXT NOT NULL,
  last_opened_at TEXT                    -- 「最近阅读」排序键
);
```

目录结构：

```
%APPDATA%\Shelf\
  library.db          （WAL 模式，附 -wal/-shm）
  books\{hash}.pdf     书籍副本
  covers\{hash}.png    封面缓存（320px 宽）
```

## 5. Tauri Commands 接口

| Command | 签名 | 说明 |
|---------|------|------|
| `add_books` | `(paths: string[]) → AddResult[]` | 哈希→查重→复制→建档；逐文件返回 `added / duplicate / error` |
| `list_books` | `(sort, query) → Book[]` | sort: `recent / added / title`；query 按书名 LIKE 过滤 |
| `remove_book` | `(id) → void` | 删记录 + 删副本与封面（不碰源文件） |
| `update_progress` | `(id, page) → void` | 同时刷新 `last_opened_at` |
| `rename_book` | `(id, title) → void` | 空标题拒绝 |
| `set_total_pages` | `(id, total) → void` | 前端打开文档后回填 |
| `save_cover` | `(hash, data: bytes) → path` | 封面 PNG 落盘并更新记录；hash 做十六进制白名单校验 |

## 6. 关键技术决策与实现要点

1. **查重以内容为准**：SHA-256 流式计算（1 MB 缓冲），同一文件改名再添加也会被识别；哈希同时用作库内文件名，天然避免重名冲突。
2. **asset 协议白名单用运行时 API**：配置文件里的 glob 作用域（`$DATA/Shelf/**`）在 Windows 反斜杠路径下匹配不可靠（曾导致 403）；改为启动时 `asset_protocol_scope().allow_directory()` 显式授权 books/covers 目录。**勿改回纯配置方案。**
3. **进度写入 = 节流 + 兜底**：翻页后 1 秒节流写库；返回书库、窗口关闭（beforeunload）强制落盘；SQLite WAL 保证已提交事务在强杀/断电后不丢。
4. **封面管线**：前端 PDF.js 渲染第 1 页 → canvas 导出 PNG → bytes 经 IPC 交给 Rust 落盘。入库时提取失败（加密/损坏）不阻塞入库，降级为「纯色渐变 + 书名」CSS 封面（色相由 hash 决定，稳定不变）；首次成功打开时自动补生成。
5. **加密 PDF 降级链**：入库成功（默认封面）→ 打开时 PDF.js 抛 `PasswordException` → 密码输入界面 → 带密码重试。
6. **异常处理原则**：单本书失败不影响批量导入（逐文件返回状态）；复制失败（如磁盘满）清理半截文件；删除时文件清理失败不阻塞记录删除。
7. **渲染正确性**：按 devicePixelRatio 放大 canvas 保证高分屏清晰；翻页/缩放时取消上一次 render task 避免竞态。

## 7. 构建与发布

```bash
npm run tauri dev     # 开发（热重载，Rust 变更自动重编译）
npm run tauri build   # 产出 NSIS 安装包
# → src-tauri/target/release/bundle/nsis/Shelf_x.x.x_x64-setup.exe
```

- 安装模式 `currentUser`：装到 `%LOCALAPPDATA%\Shelf`，免管理员权限，自动创建桌面与开始菜单快捷方式；中文安装界面。
- 发布渠道：GitHub Releases（https://github.com/Topia99/Shelf-Book-Reader/releases）。
- 无代码签名：SmartScreen 会提示「更多信息 → 仍要运行」，已写入 README FAQ。
